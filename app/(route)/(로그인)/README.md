### 로그인 로직 정리

로그인 성공 시 발급받은 `AccessToken`을 로컬 스토리지에 저장하고, 수강신청 화면으로 이동합니다.

#### 맥락

로그인 API 호출 성공 시 응답으로 `accessToken`을 받습니다.
이 토큰은 이후 인증이 필요한 모든 API 요청(예: 수강신청)의 헤더에 포함되어야 합니다.

따라서 로그인 성공 시:
1. `useAuth` 훅을 통해 토큰을 저장소(로컬 스토리지 등)에 저장합니다.
2. `useUserRole` 훅을 통해 로그인 한 사용자의 권한을 세션스토리지에 저장합니다.
3. 사용자에게 성공 토스트 메시지를 보여줍니다.
4. 수강신청이 가능한 `/courses` 페이지로 리다이렉트합니다.
